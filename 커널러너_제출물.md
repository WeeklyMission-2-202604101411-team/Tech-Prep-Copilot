# 🚀 Tech-Prep Copilot 서비스 고도화 결과 보고서

## 1. MVP 회고
### 지난주 서비스의 한계점 및 개선 필요 사항
지난주 구축한 MVP(Minimum Viable Product)는 이력서 PDF를 분석하여 직무 적합도 점수를 산출하는 핵심 기능을 성공적으로 구현하였습니다. 하지만 실제 사용 및 테스트 과정에서 다음과 같은 한계점을 발견하였습니다.

* **한계점 1. 사용자 경험(UX) 저하:** 분석 결과 및 피드백이 영문으로 출력되어 국내 사용자에게 직관적인 정보 제공이 어려움.
* **한계점 2. 정보의 최신성 부족:** 사전 구축된 RAG 데이터에만 의존하여 실시간 IT 기술 트렌드 및 최신 JD 맥락 반영 미흡.
* **한계점 3. RAG 검색 신뢰도 및 투명성 부재:** 유사도 임계값(0.3) 근거 부족으로 인한 환각 현상 및 사용자의 답변 근거 확인 불가.
* **한계점 4. 운영 효율성 및 안정성 부족:** 유료 API 단일 모델(OpenAI) 의존으로 인한 비용 부담 및 장애 시 서비스 중단 위험.

---

## 2. 서비스 고도화 부분
### 2.1 핵심 개선 및 추가 기능 설명
1. **사용성 (한국어 최적화):** 모든 시스템 프롬프트에 한국어 출력 제약 조건을 명시하고 UI를 전면 한글화하여 실제 면접 준비에 최적화된 UX 제공.
2. **기술 (검색 품질 및 최신성):** `Tavily API` 도입으로 최신 기술 동향을 반영하고, `Advanced RAG(Query Expansion)`와 실험 기반 임계값(0.4) 최적화로 환각 현상 최소화.
3. **비용 및 가용성:** `Gemini`, `Upstage` 등 멀티 모델 연동을 통한 순차적 **Fallback 아키텍처** 구축으로 운영 안정성 확보 및 비용 최적화 달성.

### 2.2 남은 한계점과 추후 개선 방향
1. **응답 캐싱(Caching) 레이어 도입:** 반복 요청에 대한 API 호출 비용 절감 및 응답 속도(Latency) 개선 필요.
2. **섀도우 로깅을 통한 모델 비교:** 실제 유저 데이터를 기반으로 멀티 모델 간의 성능(한국어 생성 능력 등) 비교 지표 확보.
3. **사용자 피드백 루프 구축:** 분석 결과 하단에 피드백 버튼을 추가하여 향후 프롬프트 튜닝을 위한 정답 데이터셋(Golden Set) 확보.
4. **멀티 모델 파싱 안정성 확보:** 모델별 응답 포맷 차이로 인한 파싱 에러를 해결하기 위해 공통 응답 인터페이스 표준화 필요.

## 3. 기술적 부분
### 3.1 전체 코드 구조 및 변경 사항
| 구분 | MVP (지난주) | 고도화 버전 (현재) | 변경 사유 |
| :--- | :--- | :--- | :--- |
| **검색 방식** | Naive RAG (단일 키워드) | Advanced RAG (멀티 쿼리 확장) | 검색 누락 방지 및 재현율 향상 |
| **데이터 필터링** | 임계값 0.3 (근거 미비) | 임계값 0.4 (실험 근거 기반) | 무관 정보 차단 및 환각 방지 |
| **모델 구조** | OpenAI 단일 모델 고정 | 멀티 모델 Fallback 아키텍처 | 서비스 가용성 및 비용 최적화 |
| **출처 신뢰도** | 출처 제목만 표시 | 제목, 링크, 유사도 등급제 도입 | 답변 근거의 투명성 제고 |
| **면접관 시스템** | 단일 면접관 | Dual-Persona (A/B 랜덤 교체) | 캐릭터별 몰입감 및 전문성 강화 |

#### 3.1.2 최종 통합 검증 결과 및 식별 이슈 (QA)
> **검증 개요:** 기획자 주도 하에 로컬 환경(Windows/VS Code) 세팅 및 멀티 모델(Gemini/Upstage) 연동을 통한 최종 통합 QA 수행 (2026.04.23)

* **성공:** RAG 기반 분석 결과 및 기술 블로그 출처 링크 정상 렌더링 확인.
* **미결 이슈 1 (페르소나):** 면접관 변경 시에도 첫 질문이 고정되는 현상 확인 (주입 로직 우선순위 조정 필요).
* **미결 이슈 2 (파싱):** 대체 모델(Gemini) 사용 시 응답 포맷 불일치로 인한 평가 데이터 산출 오류(0% 출력) 식별.

### 3.2 주요 기술 설계 의도
* **Advanced RAG:** 하나의 질문을 다양한 관점으로 확장하여 검색 재현율(Recall)을 극대화.
* **Similarity Threshold (0.4):** 실험을 통해 무관한 쿼리를 100% 차단하는 정밀도(Precision) 확보.
* **Relevance Score 시각화:** 유사도를 3단계 색상 등급으로 표현하여 AI 답변의 신뢰도를 시각적으로 전달.

### 3.3 사용 모델 및 설정값
* **Embedding Model:** BGE-M3 (ChromaDB)
* **Query Expansion:** gpt-4o-mini
* **Fallback 순서:** 1. Gemini → 2. OpenAI → 3. Upstage
* **LLM (면접 생성):** gemini-2.5-flash-lite (우선순위), gpt-4o-mini (Fallback)
